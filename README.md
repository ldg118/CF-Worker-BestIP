## 部署步骤

### 1. 准备工作

- Cloudflare 账号
- Workers KV 命名空间
- D1 数据库

### 2. 部署到 Cloudflare Workers

1. 登录 Cloudflare 控制台
2. 进入 " Workers 和 Pages "
3. 选择你的 Worker 或创建一个新的 Worker
4. 点击 "编辑代码"
5. 复制 `worker.js 或者 混淆加密` 文件的内容
6. 粘贴到代码编辑器中
7. 点击 "保存并部署"

### 3. 配置环境变量

在 Cloudflare Workers 控制台中，点击 "设置" → "变量"，添加以下环境变量：

**必需配置**：

| 环境变量             | 描述     | 示例值 |
| ---------------- | ------ | ----- |
| `ADMIN_PASSWORD` | 管理员密码 | `123` |

**可选配置**（不配置则使用默认值）：

| 环境变量                      | 描述             | 默认值 |
| ------------------------- | -------------- | ----- |
| `DEFAULT_IP_COUNT`        | 默认 IP 数量       | `3`   |
| `DEFAULT_TEST_COUNT`      | 默认测试数量         | `30`  |
| `DEFAULT_THREAD_COUNT`    | 默认线程数量         | `10`  |
| `DEFAULT_BANDWIDTH_FILE_SIZE` | 默认带宽测试文件大小   | `3`   |
| `FAILED_IP_COOLDOWN_DAYS` | 失败 IP 冷却天数     | `15`  |
| `MAX_HIGH_QUALITY_POOL_SIZE` | 最大优质池大小      | `30`  |

### 4. 配置数据库

1. **创建 D1 数据库**：
   - 进入 Cloudflare 控制台的 D1 部分
   - 创建一个新的数据库，名称为 `cf-best-ip-db`
2. **配置数据库绑定**：
   - 在 Workers 控制台中，点击 "设置" → "绑定"
   - 点击 "添加绑定"
   - 选择 "D1 数据库"
   - 绑定名称：`DB`
   - 选择你创建的 D1 数据库
   - 点击 "保存"
3. **初始化数据库表**：
   - 在 D1 数据库控制台中，运行 `migrations/001_init.sql` 中的 SQL 语句

### 5. 配置 KV 存储

1. **创建 KV 命名空间**：
   - 进入 Cloudflare 控制台的 KV 部分
   - 创建一个新的 KV 命名空间，名称为 `cf-best-ip-kv`
2. **配置 KV 绑定**：
   - 在 Workers 控制台中，点击 "设置" → "绑定"
   - 点击 "添加绑定"
   - 选择 "KV 命名空间"
   - 绑定名称：`KV`
   - 选择你创建的 KV 命名空间
   - 点击 "保存"

### 6. 配置定时任务

1. **配置 Cron 触发器**：
   - 在 Workers 控制台中，点击 "设置" → "触发器"
   - 点击 "添加 Cron 触发器"
   - 输入 cron 表达式：`0 * * * *`
   - 点击 "保存"

## 多国家 DNS 智能分流

系统支持为不同国家/地区配置不同的 DNS 域名，实现智能 IP 分流：

### 配置方法

在 Web 界面 "DNS 设置" 中配置国家域名映射：

| 国家 | 域名列表 |
|------|---------|
| CN | cf-cn.example.com, cf-china.example.com |
| US | cf-us.example.com |
| JP | cf-jp.example.com |

### 智能分流规则

- 系统根据 IP 的国家属性自动分类
- 每个国家的 IP 仅更新到对应国家的域名
- 支持一个国家配置多个域名（批量更新）
- 未匹配国家的 IP 默认更新到主域名

### 使用场景

- 为不同地区用户提供最优 IP
- 实现地理负载均衡
- 优化跨国访问速度

## 故障排除

### 常见错误

1. **DB is not defined**
   - 原因：D1 数据库绑定未配置
   - 解决：按照步骤 4 配置 D1 数据库绑定
2. **API 响应 500**
   - 原因：内部错误
   - 解决：检查 Cloudflare Workers 日志，查看具体错误信息
3. **DNS 操作失败**
   - 原因：Cloudflare API 令牌或区域 ID 错误
   - 解决：在 Web 界面 "DNS 设置" 中检查配置
4. **Telegram 通知失败**
   - 原因：Telegram 机器人令牌或聊天 ID 错误
   - 解决：在 Web 界面 "DNS 设置" → "Telegram 通知" 中检查配置

### 日志查看

1. 登录 Cloudflare 控制台
2. 进入 " Workers 和 Pages "
3. 选择你的 Worker
4. 点击 "Observability" → "Logs"
5. 查看实时日志和错误信息

## 技术架构

### 模块化结构

```
src/
├── api.js               # API 路由模块
├── config.js            # 配置管理模块
├── database.js          # 数据库操作模块
├── dns.js               # DNS 管理模块
├── errorHandler.js      # 错误处理模块

├── utils.js             # 工具函数模块
```

### 核心功能

- **IP 测试**：测试 IP 地址的响应时间和可用性
- **DNS 管理**：管理 Cloudflare DNS 记录
- **通知系统**：发送 Telegram 通知
- **数据存储**：使用 D1 数据库存储测试结果和配置
- **缓存管理**：使用 KV 存储缓存数据
- **定时任务**：定期执行 IP 测试和其他维护任务

## 安全注意事项

1. **API 令牌**：不要在代码中硬编码 API 令牌，使用环境变量
2. **密码**：使用强密码，并通过环境变量配置
3. **输入验证**：验证所有用户输入，防止注入攻击
4. **速率限制**：实现速率限制，防止滥用 API
